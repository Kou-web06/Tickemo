# App Store Connect - アプリとプライバシー / データ収集

## 概要
Tickemoで収集するデータとその目的をまとめたドキュメントです。App Store Connectの「アプリとプライバシー」セクションでの申告用です。

---

## 1. ユーザーが入力・作成するデータ

### 📸 写真・ライブ画像
- **データ種別**: メディアファイル
- **説明**: ユーザーが撮影したライブ会場の画像やチケット画像
- **収集方法**: フォトライブラリから明示的に選択
- **目的**: ライブ記録のビジュアル保存、プロフィールカードの作成
- **保存先**: デバイスのローカルストレージ（AsyncStorage、ファイルシステム）
- **第三者提供**: なし
- **リンク**: ユーザーIDと紐付け
- **用途**:
  - ◆ アプリの機能の提供
  - ○ アナリティクス
  - ○ 広告配信（非使用）

### 📝 ライブ情報入力データ
- **データ種別**: テキスト
- **説明**: 以下の情報をユーザーが入力
  - アーティスト名
  - ライブ名
  - 開催日時（YYYY.MM.DD形式）
  - 開催地（会場名）
  - 座席情報
  - 開始/終了時刻
  - QRコード（チケットのQR）
  - メモ（個人的な感想やセットリスト）
  - 詳細情報（自由記述）
- **収集方法**: アプリ内フォーム入力
- **目的**: ライブコレクションの管理とカウントダウン表示
- **保存先**: デバイスのローカルストレージ
- **第三者提供**: なし（オプション：Supabaseクラウド同期時のみ）
- **リンク**: 記録ID（UUID）で管理
- **用途**:
  - ◆ アプリの機能の提供
  - ○ アナリティクス

### 👤 プロフィール情報
- **データ種別**: テキスト、メディア
- **説明**:
  - ユーザー名（表示名）
  - アバター画像
  - SNS ハンドル（X/Twitter等）
  - 推し活開始日
  - プロフィール画像（ユーザーが設定した画像）
- **収集方法**: アプリ内設定画面で明示的に入力、フォトライブラリから画像選択
- **目的**: プロフィールカード作成、ユーザー識別
- **保存先**: デバイスのローカルストレージ
- **第三者提供**: なし（オプション：Supabaseクラウド同期時のみ）
- **リンク**: user_id（ローカルユーザー）と紐付け
- **用途**:
  - ◆ アプリの機能の提供

### 🎵 セットリスト入力データ
- **データ種別**: テキスト
- **説明**: ユーザーが記録したライブで演奏された曲のリスト
- **収集方法**: アプリ内フォーム入力
- **目的**: ライブの詳細記録
- **保存先**: デバイスのローカルストレージ（SQLite）
- **第三者提供**: なし
- **用途**:
  - ◆ アプリの機能の提供

## 2. デバイスからの自動収集データ

### 🔔 プッシュ通知トークン
- **データ種別**: テクノロジー識別子
- **説明**: Expo Notifications推送サービス用のトークン
- **収集方法**: Expo Push Notifications登録時に自動取得
- **目的**: ライブリマインダーやタイムカプセル通知の配信
- **保存先**: デバイスのローカルストレージ
- **第三者提供**: Expo（通知配信サービス）のみ
- **リンク**: デバイスに紐付け
- **用途**:
  - ◇ 通知配信
  - ◆ アプリの機能の提供

### 📱 デバイス情報
- **データ種別**: デバイスID、OSバージョン、画面仕様
- **説明**: 
  - デバイスタイプ（iOS/Android）
  - OSバージョン
  - 画面サイズ（タブレット判定用）
- **収集方法**: React Native / Expo Device APIから自動検出
- **目的**: タブレット非対応警告、UI最適化
- **保存先**: メモリ（実行時のみ）
- **第三者提供**: なし
- **用途**:
  - ◆ アプリの機能の提供

---

## 3. Apple Music連携データ

### 🎶 Apple Music認可トークン
- **データ種別**: MusicKit認可トークン
- **説明**: Apple Musicへのアクセス権限
- **収集方法**: ユーザーの明示的な許可により取得
- **目的**: アーティスト情報の検索、アルバムアートワークの取得
- **保存先**: メモリ（実行時のみ）
- **第三者提供**: Apple（MusicKit API）
- **連携API**: Apple Music Catalog API
- **用途**:
  - ◆ アプリの機能の提供

### 🎨 アーティスト情報・アルバムアート
- **データ種別**: メディア、テキスト
- **説明**: Apple Musicから取得したアーティストの画像・情報
- **収集方法**: Apple Music APIのアーティスト検索により取得
- **目的**: ランキング表示時のアーティスト画像表示
- **保存先**: メモリ（実行時キャッシュ）
- **第三者提供**: Apple（MusicKit API経由）
- **用途**:
  - ◆ アプリの機能の提供

## 4. 広告関連データ

### 📊 Google Mobile Ads
- **データ種別**: 広告ネットワーク識別子
- **説明**: 
  - Google Ads App ID
  - Ad Unit ID（バナー広告、リワード広告用）
  - AdMobユーザーID
- **収集方法**: Google Mobile Ads SDK（React Native Google Mobile Ads）
- **目的**: バナー広告とリワード広告の配信
- **保存先**: Google AdMob
- **第三者提供**: Google（AdMob）
- **設定**: 個人化されていない広告のみ
- **用途**:
  - ◆ 非個人化広告の配信
  - ○ アナリティクス

## 5. 診断データ

### 🐛 アプリケーションログ
- **データ種別**: テキストログ
- **説明**: アプリ実行時のデバッグログ、エラーログ
- **収集方法**: console.log（開発者向け）
- **目的**: アプリ動作の確認と問題診断
- **保存先**: なし（デバイスメモリのみ）
- **第三者提供**: なし
- **用途**:
  - ◆ アプリの機能提供（テストフライト時）

## 6. データ収集の許可設定と権限

### iOS 権限リスト
```
- NSPhotoLibraryUsageDescription: フォトライブラリへのアクセス
- NSPhotoLibraryAddUsageDescription: フォトライブラリへの写真保存
- NSAppleMusicUsageDescription: Apple Musicへのアクセス
```

### Android 権限リスト
```
- READ_EXTERNAL_STORAGE: 外部ストレージの読み取り
- WRITE_EXTERNAL_STORAGE: 外部ストレージへの書き込み
- READ_MEDIA_IMAGES: メディアファイルの読み取り
- READ_MEDIA_VIDEO: ビデオの読み取り
- READ_MEDIA_AUDIO: オーディオの読み取り
- READ_MEDIA_VISUAL_USER_SELECTED: 選択メディアの読み取り（Android 13+）
```

## 8. データ保持とプライバシー

### データ保持期間
- **ローカルストレージ**: ユーザーが削除するまで保持
- **Apple Music キャッシュ**: アプリ実行時のみ
- **通知トークン**: アプリ削除まで
- **ログデータ**: リアルタイムのみ（保存なし）

### データ削除機能
- **ユーザー対応**: SettingsScreen内「すべてのデータを削除」機能
- **実装**: AsyncStorage完全クリア
- **プライバシーポリシー**: https://[your-domain]/privacy（必要に応じて設定）

### クラウド同期（オプション）
- Supabase連携時のみ追加データ送信
- ユーザーのオプト選択
- サーバー側でもデータ削除可能

## 10. よくある質問

### Q. すべてのカテゴリを「個人化なし」にする理由は？
A. Tickemoはユーザーの行動を追跡してパーソナライズされたコンテンツを配信していないため。データは機能提供と統計情報取得のみに使用しています。

### Q. 「トレッキング」と「個人化」の違いは？
A. 
- **トレッキング**: ユーザーをクロスアプリ・クロスサイトで追跡することを目的とした識別子の使用
- **個人化**: ユーザーの行動パターンに基づいてコンテンツやUIをカスタマイズすること

Tickemoはどちらも行わないため、両方「いいえ」です。

### Q. Google AdMobの「個人化」は？
A. Tickemoではリワード広告に `requestNonPersonalizedAdsOnly: true` を設定しているため、個人化なし広告のみ配信しています。したがって「いいえ」です。

### Q. SuabaseクラウドSync時のデータは？
A. ユーザーが同期を**オプションで有効**にした場合のみ送信されるため、アクティブ収集として申告する必要があります（将来実装時）。

## 10. トラッキング/ATT設定

### App Tracking Transparency（iOS 14.5+）
- **ステータス**: **非対応**（トレッキング機能なし）
- **理由**: ユーザーのクロスアプリ・クロスサイト追跡を行わない
- **App Store Connect設定**: 「トレッキング」を OFF

### Android 広告ID
- **使用**: Google Mobile Ads のみ
- **個人化**: OFF（`requestNonPersonalizedAdsOnly: true`）
