# Tickemo

**ライブチケットの思い出を、一つの場所に。**

Tickemoは、アーティストのライブチケット情報を効率的に管理し、次のライブへのカウントダウンや、推し活の記録を美しく可視化するモバイルアプリです。

---

## アプリ概要

### 主な機能

- **Collection（コレクション）** - ライブチケット情報を美しいカード形式で管理
  - アーティスト名、ライブ名、日時、会場、QRコード、メモなどを保存
  - アニメーション付きのチケットカード表現
  - 複数チケットの一覧表示と詳細ビュー

- **Countdown（カウントダウン）** - 次のライブまでの日数をリアルタイム表示
  - 登録チケットから自動的に最初のライブをピックアップ
  - ティッキング表示で期待値を可視化

- **MyStage（プロフィール）** - 推し活の統計情報を可視化
  - 観戦ライブ数や来年までのカウントダウンなど、累計データを表示
  - 推し画像のカスタマイズ対応

### データの永続化

- **AsyncStorage** - チケット情報はデバイスのローカルストレージに保存
- **iCloud Sync** - iPhoneユーザーはiCloud同期に対応
- **SQLite（拡張機能）** - より高度なデータ分析用

## 🎯 制作背景

### 課題・ニーズ

推し活をしているユーザーの多くは、複数のライブチケットを紙やスマートフォンのスクリーンショットで分散管理しており、以下の課題を抱えていました：

1. **ライブ情報の分散管理** - チケットがメールやツイッター、スクリーンショットに分散している
2. **次のライブの把握困難** - 登録したチケットから「次いつ行けるのか」が一瞬でわかる仕組みがない
3. **思い出の可視化** - どのアーティストのライブに何度行ったのか、簡単には思い出せない
4. **美的価値の欠落** - 既存のチケット管理アプリは機能重視で、推し活の「楽しさ」を表現していない

### 目的

「ライブという思い出を、美しく、効率的に管理する体験」を提供することで、推し活をより楽しく、充実させることを目指しています。

## 🛠️ 技術選定およびその意図

### フレームワーク・プラットフォーム

| 技術 | 意図 |
|------|------|
| **Expo** | React Nativeの開発体験を大幅に向上させ、iOS/Android両対応を効率的に実現。プロビジョニング管理やビルド周りの複雑性を排除し、アプリロジックに集中可能。 |
| **React Native** | クロスプラットフォーム開発により、iOS/Androidの両方をTypeScriptで統一的に実装。コードベースの保守性と開発効率を向上。 |
| **TypeScript** | 型安全性により、大規模アプリの複雑なロジック（例：アニメーション制御、日付フォーマット）での バグを早期に発見。開発効率と品質を同時実現。 |

### UI・アニメーション

| 技術 | 意図 |
|------|------|
| **React Native Reanimated** | 60fpsのスムーズなアニメーション。チケットカードの回転・出入りなど、ユーザーの「楽しさ」を引き立てる表現力を実現。 |
| **React Native SVG** | ベクターグラフィックスで高解像度対応。チケットデザインなど複雑な図形をスケーラブルに描画。 |
| **Expo Linear Gradient** | グラデーション背景をネイティブレベルで高速実装。ガラスモーフィズムやモダンUI表現を実現。 |
| **React Native Pager View** | 3ページを横スワイプ遷移するタブUI。ネイティブなページング体験を提供。 |

### 状態管理・データベース

| 技術 | 意図 |
|------|------|
| **Zustand** | シンプルで予測可能な状態管理。Redux等の複雑性を排除し、チケット情報やUIの状態を直感的に管理。 |
| **AsyncStorage** | デバイスローカルへの軽量永続化。チケット情報（JSON形式）をシンプルに保存、高速にアクセス。 |
| **SQLite (expo-sqlite)** | より複雑な分析機能拡張時に備えた選択肢。リレーショナルデータが必要な場合も対応可能。 |
| **iCloud Cloudkit** | iPhoneユーザーのデバイス間でのチケット情報自動同期。ユーザーのデータロスリスク軽減。 |

### 認証・外部連携

| 技術 | 意図 |
|------|------|
| **Google Sign-In / Apple Authentication** | OSやサービスの枠を超えたシームレスなログイン。ユーザー登録の手数削減。 |
| **RevenueCat** | アプリ内購入（IAP）の管理を一元化。複数プラットフォーム間での購入データ統一。 |

### その他の技術選定

| 技術 | 用途 |
|------|------|
| **React Navigation** | 画面遷移の管理。各ページ（Countdown, Collection, MyStage）が独立したNavigationContainerを保持。 |
| **React Native ML Kit** | QRコード認識やテキスト認識。チケット情報の半自動入力を実現。 |
| **i18next + react-i18next** | 多言語対応（日本語デフォルト、将来の国際展開に対応）。 |
| **date-fns** | 日付フォーマット・計算の統一。カウントダウン正確性の確保。 |
| **Lottie React Native** | ウェルカムアニメーション等、複雑なモーションを効率的に実装。デザイナーアセットの再利用。 |

### アーキテクチャの工夫

- **Custom Floating Tab Bar** - React NavigationデフォルトのBottomTabsではなく、手作りすることで、推し活向けのカスタムUIを実現
- **RecordsContext** - AsyncStorageの読み書きを一元管理し、データの一貫性維持
- **Component-Driven Design** - TicketCard, TicketDetail等の再利用可能なコンポーネント。テスト・拡張時の負担軽減

## 🚀 開発環境セットアップ

### 必要な環境

- Node.js 20以上
- Expo CLI
- iOS開発環境（Xcode, 13以上） / Android環境（Android Studio）

### インストール・実行

```bash
# 依存関係インストール
npm install

# 開発サーバー起動
npm start

# iOS実行
npm run ios

# Android実行
npm run android

# Web実行
npm run web
```
