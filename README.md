## アプリ概要

### 主な機能

- **Collection（コレクション）** - ライブ情報を美しいチケット形式で管理
  - アーティスト名、ライブ名、日時、会場、QRコード、メモなどを保存
  - 次回のライブをカウントダウン表示
  - 複数チケットの一覧表示と詳細ビュー

- **Calendar（カレンダー）** - 次のライブまでの日数をリアルタイム表示
  - 登録したライブを日付ごとにカレンダー上で一覧表示
  - 日付タップで当日のライブ情報をすぐに確認できる

- **Report（レポート）** - 推し活の統計情報を可視化
  - 観戦ライブ数やトップアーティストなどの累計データを表示

---

## 制作背景

### 課題・ニーズ

推し活をしているユーザーの多くは、複数のライブチケットを紙やスマートフォンのスクリーンショットで分散管理しており、以下の課題を抱えていた。

1. **ライブ情報の管理** - チケットがメールや写真フォルダ、スクリーンショットなどに分散している
2. **次のライブの把握のしやすさ** - 登録したチケットから「次いつ行けるのか」が一瞬でわかる仕組みがない
3. **思い出の可視化** - どのアーティストのライブに何度行ったのか、簡単には思い出すことができない
4. **美的価値の欠落** - 既存のチケット管理アプリは機能重視で、推し活の「楽しさ」を表現しきれていない


### 目的
「ライブの思い出を、綺麗に、分かりやすく管理する体験」を提供することで、推し活をより楽しく、充実させること。

## 技術選定

### フレームワーク・プラットフォーム

| 技術 | 意図 |
|------|------|
| **Expo** | React Nativeの開発体験を大幅に向上させ、iOS/Android両対応であるため。プロビジョニング管理やビルド周りの複雑性を排除し、アプリロジックに集中できた。 |
| **React Native** | クロスプラットフォーム開発により、iOS/Androidの両方をTypeScriptで統一的に実装できるため。コードベースの保守性と開発効率を向上。 |
| **TypeScript** | 型安全性により、大規模アプリの複雑なロジック（アニメーション制御や日付フォーマットなど）でのバグを早期に発見。開発効率と品質の高さ。 |

### UI・アニメーション

| 技術 | 意図 |
|------|------|
| **React Native Reanimated** | チケットカードの開閉、ページの遷移など、ユーザーの「楽しさ」を引き立てる表現力を実現。 |
| **React Native SVG** | チケットデザインなど複雑な図形をスケーラブルに描画させるため。 |
| **Expo Linear Gradient** | グラデーション背景をネイティブレベルで高速実装。ガラスモーフィズムやモダンUI表現を実現。 |

### 状態管理・データベース

| 技術 | 意図 |
|------|------|
| **Zustand** | Reduxのボイラープレートを排除した、軽量かつ予測可能な状態管理。 |
| **AsyncStorage / SQLite** | デバイスローカルへの高速なデータ保存。将来的な複雑なクエリ（年別・アーティスト別絞り込み）を見据えた設計。 |
| **RevenueCat** | 都度課金（ホログラムチケット等の消費型アイテム）およびPlusプランのサブスクリプション管理を簡略化。 |
